03 11.1 決策表測試概述 ⭐⭐⭐

什麼是決策表測試

定義：

用於測試複雜邏輯和規則的黑箱測試技術 將條件組合與對應結果視覺化

適用場景：

✓ 銀行業務規則
✓ 保險理賠條件
✓ 考試及格標準
✓ 會員等級制度
✓ 折扣計算邏輯

共同特徵：
  → 多個條件
  → 不同組合
  → 不同結果

為什麼需要決策表

問題場景：

需求：
  IF 條件1 AND 條件2 THEN 結果A
  IF 條件1 AND NOT 條件2 THEN 結果B
  IF NOT 條件1 AND 條件2 THEN 結果C
  IF NOT 條件1 AND NOT 條件2 THEN 結果D
  
問題：
  ✗ 難以閱讀
  ✗ 容易遺漏
  ✗ 難以測試

決策表的優勢：

✓ 視覺化呈現
✓ 系統化找出所有組合
✓ 容易發現遺漏或矛盾
✓ 測試案例設計明確

04 11.2 決策表的核心元素 ⭐⭐⭐

兩大核心元素 ⭐⭐⭐ 必記

1. 條件（Condition）

別名：C, 輸入

定義：

影響系統行為的因素或判斷條件

範例：

考試場景：
  C1: 筆試成績 ≥ 85分
  C2: 實做錯誤數 ≤ 2個
  
銀行場景：
  C1: 帳戶餘額 ≥ 轉帳金額
  C2: 是否為VIP會員

2. 結果（Action/Result）

別名：A, R, 輸出

定義：

條件組合下系統應有的行為或動作

範例：

考試場景：
  A1: 通過考試
  A2: 重考
  A3: 重修
  
銀行場景：
  A1: 轉帳成功
  A2: 顯示餘額不足

條件的表示方式

布林值表示

True/False 表示：

T (True)  = 條件成立
F (False) = 條件不成立

1/0 表示：

1 = 條件成立
0 = 條件不成立

Y/N 表示：

Y (Yes) = 條件成立
N (No)  = 條件不成立

條件的撰寫技巧

明確化條件：

不推薦：
  C1: 筆試成績
    → 太模糊，如何判定成立？
  
推薦：
  C1: 筆試成績 ≥ 85分
    → 明確！1=成立, 0=不成立

05 11.3 決策規則（Decision Rule）⭐⭐⭐

什麼是決策規則

定義：

條件的特定組合及其對應的結果

別名：

  • 條件組合
  • 規則（Rule）
  • 欄位（Column）

計算決策規則數量 ⭐⭐⭐ 必背

基本公式

⚠️ 不要用 2ⁿ 公式

原因：

2ⁿ 公式前提：
  每個條件都恰好有2個可能值
  
問題：
  條件可能有 3, 4, 5... 個值
  2ⁿ 就不適用

正確公式 ⭐⭐⭐：

決策規則總數 = 各條件可能值數量相乘

C1有m個值
C2有n個值  → 總規則數 = m × n × p
C3有p個值

範例計算

範例1：全是2值

C1: 天氣好/不好（2個值）
C2: 有錢/沒錢（2個值）
C3: 心情好/不好（2個值）

規則數 = 2 × 2 × 2 = 8

範例2：混合值

C1: 學歷（5個值：國小、國中、高中、大學、碩士）
C2: 有證照/無證照（2個值）
C3: 經驗年資（3個值：<2年、2-5年、>5年）

規則數 = 5 × 2 × 3 = 30

範例3：考試題型

題目：有4個條件
  C1: 2個可能值
  C2: 2個可能值
  C3: 3個可能值
  C4: 2個可能值
  
問：共有幾個決策規則？

答案：2 × 2 × 3 × 2 = 24

與結果的關係 ⭐⭐⭐

重要觀念：

決策規則數量與結果數量無關

為什麼？

決策規則 = 條件組合
  ↓ 只看
條件的數量和可能值

結果的數量：
  只影響每個規則的輸出
  不影響規則總數

範例：

3個條件，各2個值 → 8個規則
  
無論結果有：
  1個結果（全部通過/不通過）
  2個結果（通過/不通過）
  8個結果（每個規則不同結果）
  
規則數永遠是 8 個

06 11.4 決策表製作方法 ⭐⭐⭐

製表標準流程

步驟1：識別條件與結果

從需求中找：

需求範例：
  "考生筆試成績超過85分，且實做最多錯2個，
   則通過考試。若其中一個沒過，需重考該項。
   若兩個都沒過，需重修。"
  
識別：
  條件：
    C1: 筆試成績 ≥ 85
    C2: 實做錯誤數 ≤ 2
  
  結果：
    A1: 通過
    A2: 重考
    A3: 重修

步驟2：列出所有條件組合

方法1：真值表法（適合程式背景）

2個條件，各2值 = 4個規則

R1  R2  R3  R4
1   1   0   0  ← C1
1   0   1   0  ← C2

規律：

第1個條件：對半分
  → 1 1 0 0

第2個條件：各半再對半
  → 1 0 1 0

第3個條件：更細分
  → 1 0 1 0 1 0 1 0

方法2：分組法（更直觀）

步驟：
1. 先寫總規則數（如8條）
2. 第1個條件：對半
   → 前4個為1，後4個為0
3. 第2個條件：各半再對半
   → 1100 1100
4. 第3個條件：繼續細分
   → 10101010

3條件範例：

    R1 R2 R3 R4 R5 R6 R7 R8
C1: 1  1  1  1  0  0  0  0
C2: 1  1  0  0  1  1  0  0
C3: 1  0  1  0  1  0  1  0

步驟3：填入對應結果

依需求填入：

每個規則組合下
  → 查看需求
  → 確定對應結果
  → 填入

範例：
  R1(1,1,1) → 查需求 → 結果A
  R2(1,1,0) → 查需求 → 結果B
  ...

製表示範

完整範例：上課決策

情境：

決定是否來上課

條件：
  C1: 天氣好（1=好, 0=不好）
  C2: 心情好（1=好, 0=不好）
  C3: 有錢（1=有, 0=沒有）
  
結果：
  A1: 來上課
  A2: 去睡覺

決策表：

       R1  R2  R3  R4  R5  R6  R7  R8
C1天氣  1   1   1   1   0   0   0   0
C2心情  1   1   0   0   1   1   0   0
C3有錢  1   0   1   0   1   0   1   0
---------------------------------------
A1上課  0   0   0   01   1   0   1   0
A2睡覺  0   0   0   0   0   0   1   1

註：1=做, 0=不做

解讀規則：

R1(1,1,1): 天氣好+心情好+有錢 → 不上課，不睡
R2(1,1,0): 天氣好+心情好+沒錢 → 不上課，不睡
R3(1,0,1): 天氣好+心情不好+有錢 → 不上課，不睡
R4(1,0,0): 天氣好+心情不好+沒錢 → 不上課，不睡
R5(0,1,1): 天氣不好+心情好+有錢 → 上課
R6(0,1,0): 天氣不好+心情好+沒錢 → 上課
R7(0,0,1): 天氣不好+心情不好+有錢 → 不上課，要睡
R8(0,0,0): 天氣不好+心情不好+沒錢 → 不上課，要睡

07 11.5 決策表的靜態測試價值 ⭐⭐⭐

製表即審查

重要觀念：

製作決策表的過程 = 靜態測試

為什麼？

製表時會發現：
  ✓ 需求不完整
  ✓ 規則矛盾
  ✓ 條件遺漏
  ✓ 結果不明確

發現問題範例

問題1：規則遺漏

情境：

需求只寫：
  "筆試≥85且實做≤2個錯誤 → 通過"
  "其中一個沒過 → 重考"
  
製表時發現：
  兩個都沒過怎麼辦？
    ↓ 需求沒寫！

解決：

回去詢問需求提供者
  → 補充規則
  → 更新需求文件

問題2：規則矛盾

範例：

R2和R3的條件組合相同
  但結果不同
    ↓ 矛盾！
    
R2: (F, F, F) → 重考
R3: (F, F, F) → 重修

怎麼辦？
  → 與需求方確認
  → 修正矛盾

問題3：條件不明確

模糊條件：

需求："無論性別"
  
問題：
  如何在決策表中表達"無論"？

解決方案：

方法1：拆分條件
  C1: 性別 = 男
  C2: 性別 = 女
  
  這樣可表達：
    (0, 0) = 無論性別
    (1, 0) = 男性
    (0, 1) = 女性

08 11.6 從決策表設計測試案例 ⭐⭐⭐

核心原則 ⭐⭐⭐ 必記

測試案例與規則關係：

每個決策規則至少對應一個測試案例

為什麼？

每個規則代表一種情境
  → 每種情境都要測試
  → 確保完整性

設計流程

步驟1：決策表完成

條件和結果都已填入
所有規則都已明確

步驟2：為每個規則設計測試案例

設計原則：

每個規則 → 找符合條件的值

範例：
  R1: C1=T, C2=T
    → 需要：C1成立的值 + C2成立的值

完整設計範例

範例：考試及格判定

決策表：

           R1  R2  R3  R4
筆試≥85   T   T   F   F
實做≤2錯  T   F   T   F
--------------------------
通過      X
重考筆試      X
重考實做          X
重修                  X

測試案例設計：

TC1 命中 R1(T,T)：
  筆試：85分（≥85成立）
  實做：2個錯（≤2成立）
  期望：通過 ✓

TC2 命中 R2(T,F)：
  筆試：95分（≥85成立）
  實做：3個錯（≤2不成立）
  期望：重考實做 ✓

TC3 命中 R3(F,T)：
  筆試：84分（≥85不成立）
  實做：1個錯（≤2成立）
  期望：重考筆試 ✓

TC4 命中 R4(F,F)：
  筆試：70分（≥85不成立）
  實做：3個錯（≤2不成立）
  期望：重修 ✓

進階：結合邊界值分析 ⭐⭐

可以更精確取值：

原本 R2：
  筆試：95（任意≥85的值）
  實做：3（任意>2的值）
  
改用邊界值：
  筆試：85或86（邊界附近）
  實做：3（邊界外最近值）
    ↓ 更有機會
  發現邊界錯誤

混用技巧：

決策表 → 確定要測哪些組合
  +
邊界值 → 優化每個組合的取值
  =
更強的測試設計

09 11.7 決策表覆蓋率 ⭐⭐⭐

覆蓋率定義 ⭐⭐⭐ 必背

公式：

決策表覆蓋率 = 測試的規則數 / 總規則數 × 100%

分母：決策規則總數 分子：測試案例覆蓋的規則數

覆蓋率計算範例

範例1：基本計算

決策表：

共 8 個規則（R1-R8）

測試狀況：

TC1 → 覆蓋 R1
TC2 → 覆蓋 R3
TC3 → 覆蓋 R5
  
覆蓋率 = 3/8 = 37.5%

範例2：重複覆蓋

測試狀況：

TC1 → 覆蓋 R2
TC2 → 覆蓋 R2（重複）
TC3 → 覆蓋 R5
  
覆蓋率 = 2/8 = 25%

注意：
  R2雖被測2次
  但只算1個規則

範例3：100%覆蓋

要求：

每個規則都至少被測1次

實現：

8個規則 → 至少8個測試案例
  
TC1 → R1
TC2 → R2
TC3 → R3
TC4 → R4
TC5 → R5
TC6 → R6
TC7 → R7
TC8 → R8
  
覆蓋率 = 8/8 = 100% ✓

10 11.8 可行性分析與簡化 ⭐⭐⭐

可行的決策規則

概念：

並非所有理論組合都會在現實發生

範例：體檢場景

條件：
  C1: 做體檢（T/F）
  C2: 量血壓（T/F）
  C3: 測BMI（T/F）
  
理論上：2^3 = 8種組合

但實際：
  (F, T, F) → 不做體檢但量血壓？
  (F, F, T) → 不做體檢但測BMI？
  (F, T, T) → 不做體檢但又量又測？
    ↓ 不可能！

可行規則：

只有：
  (T, T, T) → 做體檢+量血壓+測BMI ✓
  (T, T, F) → 做體檢+量血壓 ✓
  (T, F, T) → 做體檢+測BMI ✓
  (T, F, F) → 做體檢 ✓
  (F, F, F) → 都不做 ✓
  
剩下3個不可行 → 可刪除

簡化決策表

簡化（Simplification）：

刪除不可行的規則

前提條件 ⭐⭐⭐：

必須確定該規則：
  ✓ 真實世界不會發生
  ✓ 有明確證據支持
  
不能隨意刪除

簡化效果：

原本：8個規則 → 8個測試案例
  ↓ 簡化
刪掉3個 → 5個測試案例
  
節省37.5%測試工作量

合併規則（最小化）

概念（進階級內容）：

合併結果相同的規則

條件：

只能合併結果完全相同的規則

表示方式：

使用符號：
  - （破折號）
  ~ （波浪號）
  
表示：該條件真或假都可

範例：

R4: (T, T, F) → 通過
R5: (T, T, T) → 通過
  ↓ 合併
R4': (T, T, -) → 通過

意思：
  只要 C1=T, C2=T
  C3無論T或F
  都通過

簡化風險 ⚠️：

合併後：
  規則數減少
  測試案例減少
  
風險：
  可能漏測某些組合
  
範例：
  (T,T,-) 代表 (T,T,T) 和 (T,T,F)
  如果只測 (T,T,T)
  可能漏掉 (T,T,F) 的 Bug

11 11.9 實戰案例分析

案例1：網路銀行轉帳

需求：

即時轉帳規則：
  1. 餘額≥轉帳金額 → 轉帳成功
  2. 餘額<轉帳金額 → 顯示餘額不足

決策表：

           R1  R2
餘額≥金額  T   F
------------------
轉帳成功   X
餘額不足       X

測試案例：

TC1（R1）：
  餘額：1000, 轉帳：500
  期望：轉帳成功，顯示成功訊息
  
TC2（R2）：
  餘額：500, 轉帳：1000
  期望：轉帳失敗，顯示餘額不足

案例2：優惠折扣計算

需求：

會員折扣規則：
  - 白金會員：20%折扣
  - 黃金會員：10%折扣
  - 一般會員：無折扣
  - 折扣日另+5%

條件識別：

C1: 會員等級（3個值：白金、黃金、一般）
C2: 是否折扣日（2個值：是、否）

規則數 = 3 × 2 = 6

決策表：

          R1  R2  R3  R4  R5  R6
等級      白  白  黃  黃  般  般
折扣日    Y   N   Y   N   Y   N
----------------------------------------
基本折扣  20% 20% 10% 10% 0%  0%
額外折扣  5%  0%  5%  0%  5%  0%
總折扣    25% 20% 15% 10% 5%  0%

案例3：汽車保險費率

需求（簡化版）**：

保費計算：
  年齡：18-65歲
  吸菸：是/否
  地區：台北/非台北
  
規則：
  - <18或>65：不承保
  - 18-65 + 吸菸 + 台北：標準費率×1.5
  - 18-65 + 吸菸 + 非台北：標準費率×1.3
  - 18-65 + 不吸菸 + 台北：標準費率×1.0
  - 18-65 + 不吸菸 + 非台北：標準費率×0.9

決策表（部分）：

           R1   R2   R3   R4   R5
年齡18-65  T    T    T    T    F
吸菸       T    T    F    F    -
台北       T    F    T    F    -
-----------------------------------------
費率       1.5  1.3  1.0  0.9  不承保

測試設計：

TC1（R1）：32歲吸菸台北 → 費率1.5
TC2（R2）：45歲吸菸高雄 → 費率1.3
TC3（R3）：28歲不吸台北 → 費率1.0
TC4（R4）：50歲不吸新竹 → 費率0.9
TC5（R5）：17歲 → 不承保

12 11.10 考試題型與解題技巧 ⭐⭐⭐

題型1：規則數量計算（K1）

標準題型：

Q: 有3個條件，分別有2,3,2個可能值，
   共有幾個決策規則？

解法：

2 × 3 × 2 = 12個規則

題型2：覆蓋率計算（K3）

題目範例：

決策表有5個規則
已有測試：
  TC1 → R4
  TC2 → R4（重複）
  TC3 → R2
  
覆蓋率為？
