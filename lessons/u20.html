03 20.1 測試工具分類 ⭐⭐

重要提醒

工具分類沒有統一規範！

考試重點：
  理解各類工具的定義
  不是背特定工具名稱

九大類工具（官方分類）

1. 管理類工具：

目的：
  促進測試流程效率與有效性
  
範例：
  ✓ 需求管理
  ✓ 測試管理
  ✓ 缺陷管理
  ✓ 構型管理

2. 靜態測試工具：

支援審查與靜態分析

3. DevOps工具：

✓ CI/CD工具
✓ 工作流程追蹤
✓ 自動化建置

4. 協作工具：

促進溝通

5. 擴展性與部署工具：

✓ 虛擬機
✓ 容器化工具

6. 測試執行與覆蓋率工具：

自動執行測試
量測覆蓋率

7. 測試設計與實作工具：

產生測試案例
測試數據
測試程序

8. 非功能性測試工具：

效能測試
安全測試等

9. 資料準備工具：

支援測試數據準備

04 20.2 測試自動化效益 ⭐⭐⭐

六大效益（必考）

1. 節省時間
   - 重複性工作交給工具

2. 一致性
   - 測試更穩定

3. 客觀評估
   - 減少人為偏差

4. 易於收集資料
   - 電腦化記錄

5. 減少測試時間
   - 有時間做其他事

6. 複雜度量
   - 工具可提供人類難以計算的度量

05 20.3 測試自動化風險 ⭐⭐⭐

常見風險（必考）

1. 不切實際的期待
   - 買工具≠成功

2. 導入成本
   - 需要時間訓練
   - 需要整合

3. 適合性問題
   - 有些測試適合手動
   - 不能過度依賴工具

4. 供應商問題
   - 工具淘汰
   - 廠商倒閉

5. 相容性問題
   - 與開發平台不相容

6. Open Source風險
   - 社群支援不確定

7. 維護成本
   - 測試程式需要維護

06 20.4 自動化建議 ⭐⭐

高投資報酬率

ISTQB建議優先自動化：

1. 迴歸測試 ⭐⭐⭐
   - 重複執行
   - 頻率高
   
2. 煙霧測試
   - 核心功能
   - 每次都要跑

自動化原則

不是所有測試都要自動化！

考慮：
  ✓ 是否重複執行？
  ✓ 是否容易自動化？
  ✓ 投資報酬率？
  ✓ 維護成本？

07 20.5 工具選擇考量 ⭐

選擇因素

✓ 符合組織需求
✓ 與開發平台相容
✓ 易於整合
✓ 供應商支援
✓ 成本與效益
✓ 是否需要訓練

09 20.7 構型管理 ⭐⭐

定義

機制：
  識別、控制、追蹤工作產品
  
目的：
  版本控制
  可追溯性

支援測試

✓ 唯一識別所有版本
✓ 追蹤所有變更
✓ 需求明確參照
✓ 可回溯到先前版本

重點：

構型管理 ≠ 缺陷管理

缺陷狀態 → 缺陷管理處理
版本控制 → 構型管理處理
