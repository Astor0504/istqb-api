01 單元18：風險管理基礎

02 學習目標

  • 理解風險的定義與組成
  • 區分產品風險與專案風險
  • 掌握風險評估方法
  • 了解風險應對策略

03 18.1 風險定義 ⭐⭐⭐（必背）

風險的兩大要素

風險 = 發生機率 × 影響程度

兩者缺一不可！

範例：

機率：高、中、低
影響：高、中、低

或

機率：0.8
影響：$10,000

04 18.2 風險分類 ⭐⭐⭐（必考）

兩大類風險

1. 產品風險（品質風險）：⭐⭐⭐

定義：
  與產品品質特徵相關的風險
  
範例：
  ✓ 功能遺漏
  ✓ 計算錯誤
  ✓ 回應時間太慢
  ✓ 安全漏洞
  ✓ 使用者體驗差
  ✓ 架構不佳

2. 專案風險（規劃風險）：

定義：
  與專案管理控制相關的風險
  
範例：
  ✓ 進度延誤
  ✓ 預算不足
  ✓ 人員流失
  ✓ 工具支援不足
  ✓ 範疇蔓延

考試重點 ⭐⭐⭐

題型：

「下列何者非產品風險？」

解題：
  只要記住產品風險定義
  其他都是專案風險

測試只能解決：

產品風險 ✓
專案風險 ✗

05 18.3 風險評估 ⭐⭐

定性評估（矩陣法）

風險矩陣：

          影響程度
        低    中    高
機  高  中    高    高
率  中  低    中    高
    低  低    低    中

優點：

✓ 不需精確數字
✓ 容易使用
✓ 符合實際

定量評估（數值法）

計算公式：

風險等級 = 機率 × 影響

範例：

風險A：0.1 × 0.3 = 0.03
風險B：0.4 × 0.6 = 0.24
風險C：0.8 × 0.5 = 0.40

排序：C > B > A

06 18.4 風險應對策略 ⭐⭐⭐

四種策略（必背）

1. 接受：

承擔風險
不做特別處理

2. 轉移：

轉嫁給他人
如：買保險

3. 應變計劃：

事先規劃
萬一發生時的對策

4. 舒緩（測試的角色）：⭐⭐⭐

降低發生機率
或降低影響程度

測試主要用於舒緩！

測試如何舒緩產品風險

✓ 選擇適當的測試人員
✓ 應用適當的獨立性
✓ 執行審查或靜態分析
✓ 選擇適當的測試技術
✓ 選擇適當的覆蓋等級
✓ 針對品質特徵選擇測試類型

07 18.5 風險導向測試 ⭐⭐

核心概念

高風險 → 投入更多測試
低風險 → 投入較少測試

實施步驟

1. 識別風險
2. 評估風險等級
3. 排序測試案例
4. 分配測試資源

08 18.6 考試重點整理 ⭐⭐⭐

必背內容

1. 風險定義：

機率 × 影響

2. 風險分類：

產品風險（測試可解決）
專案風險（測試不能解決）

3. 應對策略：

接受、轉移、應變、舒緩
測試 = 舒緩

答題技巧

產品風險判斷：

問自己：
  這是品質特徵問題嗎？
  
是 → 產品風險
否 → 專案風險

09 課後練習

練習1：判斷風險類型

a) 系統回應時間過長
b) 測試人員經驗不足
c) 安全漏洞
d) 預算削減

產品風險：a, c
專案風險：b, d

練習2：舒緩策略

風險：購物車計算錯誤
機率：高
影響：高

舒緩方法：
  □ 應用等價劃分設計測試
  □ 應用邊界值分析
  □ 執行決策表測試
  □ 增加測試覆蓋率
