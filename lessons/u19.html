01 單元19：測試監控與報告

02 學習目標

  • 理解測試監控的目的
  • 掌握測試報告的類型與欄位
  • 了解度量指標的應用
  • 學會報告撰寫重點

03 19.1 測試監控 ⭐⭐

監控 = 監督 + 控制

監督（Monitoring）：

收集資料
了解測試現況

控制（Control）：

採取管理策略
改善品質

監控目的

✓ 進度追蹤
✓ 品質評估
✓ 目標檢查

04 19.2 測試進度報告 ⭐⭐⭐（必背欄位）

報告欄位（需背）

第五章會詳細說明6-8個欄位

核心：
  ✓ 測試進度
  ✓ 缺陷狀況
  ✓ 覆蓋率
  ✓ 風險狀況

報告時機

定期產出
持續更新
供利害關係人決策

05 19.3 測試總結報告 ⭐⭐⭐（必背欄位）

報告欄位（需背）

第五章會詳細說明

核心：
  ✓ 測試摘要
  ✓ 測試結果
  ✓ 缺陷統計
  ✓ 經驗教訓
  ✓ 後續建議

報告時機

測試完成階段產出
專案結束時
迭代結束時

06 19.4 報告撰寫原則 ⭐⭐

核心原則

讓目標讀者看得懂！

報告對象不同 → 重點不同

老闆喜歡：
  圖表、摘要

工程師喜歡：
  詳細數據、細節

溝通方式

✓ 口頭報告
✓ 儀表板（Dashboard）
✓ Email
✓ 即時聊天
✓ 線上文件
✓ 正式報告

選擇依據：

分散式團隊 → 正式溝通
集中式團隊 → 可較非正式

07 19.5 度量指標 ⭐⭐

三大應用

1. 進度追蹤
   - 任務完成度
   - 測試案例執行狀況

2. 品質評估
   - 缺陷數量
   - 缺陷密度

3. 目標檢查
   - 覆蓋率
   - 風險等級

重要提醒 ⭐⭐⭐

重點是公式，不是數字！

錯誤：
  "缺陷密度0.3很厲害"
  
正確：
  先看公式定義
  - 分母是什麼？
  - 分子是什麼？
  
公式不同，數字無法比較

範例：

缺陷密度 = 缺陷數量 / 軟體規模

問題：
  軟體規模用程式碼行數？
  還是功能點數？
  
  缺陷包含全部階段？
  還是特定階段？

08 19.6 考試重點 ⭐⭐⭐

必考內容

1. 兩種報告：

進度報告（定期）
總結報告（完成時）

兩者欄位要分清楚

2. 報告原則：

讓目標讀者看得懂

3. 度量指標：

理解定義
不是記數字

09 19.7 測試計劃書 ⭐⭐⭐

重要性

五星上將名言：

"計劃沒有用，但計劃必不可少"

為什麼？
  強迫提早思考問題

計劃書欄位（必背）

來自ISO 29119：

✓ 測試時程
✓ 引入/引出標準
✓ 風險清單
✓ 測試範圍
✓ 測試策略
✓ 資源需求
✓ 度量指標
（第五章會詳細說明）

10 19.8 引入/引出標準 ⭐⭐

引入標準（Entry Criteria）

定義：
  進入活動前
  必須具備的條件
  
範例：
  ✓ 需求文件完成
  ✓ 測試環境就緒
  ✓ 測試數據準備完成

引出標準（Exit Criteria）

定義：
  結束活動
  必須達到的目標
  
範例：
  ✓ 達到覆蓋率目標
  ✓ 高優先缺陷都修復
  ✓ 預算用完
  ✓ 時間用完

特別提醒：

預算/時間用完 → 可以是引出標準
  
但不能只用這個！
還需要：
  ✓ 風險評估報告
  ✓ 詳細測試報告
  ✓ 給決策者資訊

11 課後練習

練習1：區分報告類型

哪個是進度報告內容？
a) 經驗教訓
b) 測試執行狀況
c) 後續建議
d) 測試摘要

答案：b（其他是總結報告）

練習2：引入vs引出

判斷以下是引入還是引出標準：

a) 測試環境就緒
b) 達到80%覆蓋率
c) 預算批准
d) 所有高優先缺陷修復

引入：a, c
引出：b, d
