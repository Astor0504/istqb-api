03 1.1 ISTQB 認證簡介

什麼是 ISTQB？

  • 全球領先的軟體測試認證組織
  • 類似專案管理領域的 PMI（推動 PMP 認證）
  • 非營利組織，不綁定特定工具
  • 在 120+ 國家推行認證制度

為什麼需要 ISTQB 認證？

個人價值

  1. 厚實基礎知識：系統化學習測試專業知識
  2. 專業能力肯定：國際認可的專業證明
  3. 職涯發展：有助於轉職與晉升
  4. 終身有效：證書永久有效，無需續費或累積學分

企業價值

  1. 對外證明：向國際客戶證明 QA 團隊專業能力
  2. 團隊提升：建立共同的測試知識語言
  3. 品質保證：確保測試流程符合國際標準

04 1.2 ISTQB 證照體系

證照層級結構

Expert Level（專家級）
    ↑
Advanced Level（進階級）
    ↑
Foundation Level（基礎級）← 本課程重點

Foundation Level 的重要性

  • 入門必考：所有進階證照的基礎
  • 知識核心：涵蓋軟體測試基本概念
  • 應用廣泛：適用於所有測試層級

進階發展路徑

Core（核心）

  • Test Analyst（TA）- 黑箱測試專家
  • Technical Test Analyst（TTA）- 白箱測試專家
  • Test Manager（TM）- 測試管理專家

Agile（敏捷）

  • Agile Technical Tester
  • Agile Test Leadership

Specialist（專業領域）

  • AI Testing（AI 測試）
  • Automotive Software Testing（車載軟體測試）
  • Game Testing（遊戲測試）
  • Mobile Application Testing（行動應用測試）
  • Performance Testing（效能測試）
  • Security Testing（安全測試）

05 1.3 考試規則與要求

考試基本資訊

項目內容
題目數量40 題
題型單選題
考試時間60 分鐘（母語）<br>75 分鐘（英文）
通過標準65%（26 題正確）
計分方式1 題 = 1 分
證書效期終身有效

題目難度分級（K-Level）

K1：記憶型題目

  • 考核：記憶與辨識
  • 範例：「ISO 29119 是什麼標準？」
  • 準備方式：直接背誦

K2：理解型題目

  • 考核：理解與比較
  • 範例：「靜態測試與動態測試的差異？」
  • 準備方式：理解概念

K3：應用型題目

  • 考核：實際應用能力
  • 範例：「用邊界值分析設計測試案例」
  • 準備方式：大量練習

章節題數K1K2K3難度

考試策略重點：

  • 第4章最重要（11題），需投入最多準備時間
  • 第6章最簡單（2題），快速複習即可
  • 前3章為送分題，務必掌握

06 1.4 語言選擇建議

繁體中文 vs 英文

繁體中文優勢

✓ 題意理解更直接 ✓ 減少語言障礙 ✓ 考試時間更充裕 ✓ 降低誤解風險

英文考試特點

  • 詞彙程度：約大學英文水平
  • 題目長度：平均 2-3 行
  • 專業術語：需事先熟悉
  • 額外時間：+15 分鐘

建議：除非英文能力特別優秀，否則建議選考繁體中文

07 1.5 報考流程

Step 1：選擇考試機構

主要考試平台

1. Pearson VUE

  • 優點：考場最多（全台灣各縣市）
  • 方式：可選實體考場或線上考試
  • 費用：約 6,000-7,000 台幣

2. iSQI（推薦考繁體中文）

  • 優點：提供繁體中文考試
  • 優惠：可申請折扣（約 25 美金優惠）
  • 方式：實體/線上皆可

3. GASQ

  • 優點：有保險方案（重考優惠）
  • 適合：對第一次考試沒把握者

Step 2：預約考試

重要提醒：

  1. 提前至少1個月預約
  • 確保有合適的考場
  • 確保有合適的時段
  1. 選擇考場建議
  • 選擇離家近的考場
  • 確認交通便利性
  • 避開偏遠地點
  1. 實體考場 vs 線上考試
比較項目實體考場線上考試
環境控制✓ 穩定✗ 需自備密閉空間
設備需求✓ 考場提供✗ 需電腦+手機
輔助工具✓ 提供草稿紙筆✗ 僅線上記事本
干擾風險✓ 低✗ 高（家人、寵物）
推薦度⭐⭐⭐⭐⭐⭐⭐☆☆☆

強烈建議：選擇實體考場

Step 3：準備考試

（詳見各單元學習內容）

Step 4：參加考試

  • 提前15-30分鐘到達
  • 攜帶有效身份證件
  • 放鬆心情應考

Step 5：查詢成績

  • 考試當天即可知道結果
  • 隔天收到正式email通知
  • 約1個月後可在官網查詢認證

10 1.8 必背重點清單

標準編號（必考）

  • ✓ ISO 29119：軟體測試系列標準
  • ✓ ISO 25010：軟體產品品質特徵
  • ✓ ISO 20246：工作產品審查標準

品質特徵（9個）

  1. Functional Suitability（功能適用性）
  2. Performance Efficiency（效能效率）
  3. Compatibility（兼容性）
  4. Usability（可用性）
  5. Reliability（可靠性）
  6. Security（安全性）
  7. Maintainability（可維護性）
  8. Portability（可攜性）
  9. Flexibility（彈性）

測試層級（5個）

  1. Unit Testing（單元測試）
  2. Component Integration Testing（元件整合測試）
  3. System Integration Testing（系統整合測試）
  4. System Testing（系統測試）
  5. Acceptance Testing（驗收測試）

測試活動（7個，需記順序）

  1. Test Planning（測試規劃）
  2. Test Analysis（測試分析）
  3. Test Design（測試設計）
  4. Test Implementation（測試建置）
  5. Test Execution（測試執行）
  6. Test Monitoring and Control（測試監控）
  7. Test Completion（測試完成）

注意：監控活動持續進行，排序題通常不包含

11 1.9 考試技巧

答題策略

  1. 時間分配：每題平均 1.5 分鐘
  2. 先易後難：先完成有把握的題目
  3. 刪去法：善用選項回推
  4. 標記系統：標記不確定的題目

常見陷阱

⚠️ 數字混淆：標準編號、百分比 ⚠️ 術語相似：Defect vs Failure vs Error ⚠️ 選項干擾：3.1 版本內容（已淘汰） ⚠️ 邏輯題：需仔細讀題與畫圖

必過門檻心理建設

  • 26 題及格，不是 40 題滿分
  • 證書不顯示分數
  • 重點是聰明地通過，不是追求滿分
  • 時間夠用，無需過度緊張

⭐ 學習建議

  1. 系統化學習：按單元順序學習
  2. 理解為主：不要死記硬背
  3. 大量練習：特別是第4章
  4. 定期複習：避免遺忘
  5. 善用資源：充分利用題庫

13 1.11 AI 時代的測試挑戰

為什麼測試更重要？

  • AI 可以快速開發程式碼
  • 但 AI 生成的程式碼需要驗證
  • AI 軟體可能產生幻覺（看似正確實則錯誤）
  • 測試無法完全自動化，需要人工判斷

測試人員的價值

「快速開發 + 沒有測試 = 快速死亡」
  • 確保 AI 生成程式碼的正確性
  • 驗證系統行為符合預期
  • 降低軟體失敗風險
  • 保護使用者與企業利益

15 附錄：常見問題

Q1：一定要上課才能考試嗎？ A：不用，ISTQB 不強制上課，可自行準備。

Q2：證書會過期嗎？ A：不會，終身有效。

Q3：考試可以重考嗎？ A：可以，但需重新報名與繳費。

Q4：重考有次數限制嗎？ A：沒有限制。

Q5：考過後可以查詢嗎？ A：可以在官網公開查詢系統查詢（需本人同意公開）。
