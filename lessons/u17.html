01 單元17：測試案例排序策略

02 學習目標

  • 理解測試案例排序的重要性
  • 掌握三種主要排序策略
  • 理解優先性與依賴性的關係
  • 學會選項回推法解題

03 17.1 排序的重要性 ⭐⭐

為什麼需要排序？

現實限制：

情況：
  100個測試案例
  只有3天時間
  不可能全部執行
  
問題：
  先測哪些？
  
解決：
  需要排序策略

排序目標

✓ 優先發現重要缺陷
✓ 優先覆蓋關鍵功能
✓ 優先降低高風險
✓ 最大化測試價值

04 17.2 三種排序策略 ⭐⭐⭐（必背）

策略1：基於風險

核心：

高風險 → 優先測試

範例：

TC1: 登入（高風險）
TC2: 查詢（低風險）
TC3: 支付（高風險）

排序：TC1, TC3, TC2

策略2：依覆蓋率

核心：

高覆蓋率 → 優先測試

增量覆蓋：

選能新增最多覆蓋的測試案例

策略3：依需求重要性

核心：

重要需求 → 優先測試

05 17.3 處理依賴關係 ⭐⭐⭐

依賴 vs 優先性

核心原則：⭐⭐⭐

依賴關係 > 優先性
  
原因：
  不滿足依賴 = 無法執行

解題範例

題目：

測試案例    優先性    依賴
TC001       3         無
TC002       2         TC001
TC003       1         TC002
TC004       2         TC002
TC005       3         TC002

問：第三個執行誰？

分析：

TC003優先性最高（1）
但依賴TC002
TC002依賴TC001

必須：TC001 → TC002 → ...

TC002後可執行：
  TC003（優先性1）
  TC004（優先性2）
  TC005（優先性3）
  
選TC003（優先性最高）

答案：第三個是TC003

06 17.4 選項回推法 ⭐⭐⭐（必學）

為什麼要回推？

正推問題：
  ✗ 表格複雜
  ✗ 容易看錯
  ✗ 關係多

回推優點：
  ✓ 從答案驗證
  ✓ 快速排除
  ✓ 不易出錯

解題步驟

Step 1：看第一個位置

所有選項第一個是誰？
  
找出哪些不可能第一個
→ 刪除選項

Step 2：看後續位置

剩餘選項
逐一驗證
繼續刪除

Step 3：確認答案

最後剩下的就是答案

實戰範例 ⭐⭐⭐

題目：

TC1: 優先性1，依賴TC2
TC2: 優先性2，無依賴
TC3: 優先性1，依賴TC2  
TC4: 優先性3，無依賴

選項：
a) TC1 → TC2 → TC3 → TC4
b) TC2 → TC1 → TC3 → TC4
c) TC4 → TC2 → TC1 → TC3
d) TC3 → TC2 → TC1 → TC4

解答：

Step 1：第一個可能是誰？
  
TC1依賴TC2 → TC1不能第一個
TC3依賴TC2 → TC3不能第一個
  
→ 刪除選項a, d

剩下：b, c

Step 2：比較b和c
  
選項b：TC2第一個
  已TC2無依賴，優先性2 ✓
  
選項c：TC4第一個
  TC4無依賴，但優先性3（最低）
  TC2優先性2較高
  → TC2應優先
  
→ 刪除選項c

答案：b

07 17.5 考試重點 ⭐⭐⭐

必背

1. 三種策略：

✓ 風險
✓ 覆蓋率  
✓ 需求重要性

2. 核心原則：

依賴 > 優先性

3. 解題method：

選項回推法

答題技巧

1. 先畫依賴圖
2. 找無依賴的
3. 用選項驗證
4. 快速刪除

08 課後練習

練習題：

測試案例    優先性    依賴
TC1         2         無

第二個執行誰？

提示：

TC1必須第一个
TC2和TC3可能第二個
看優先性都是1
→ 兩個都可以
