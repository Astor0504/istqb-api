03 10.1 為什麼需要邊界值分析 ⭐⭐⭐

小強理論（蟑螂理論）

核心概念：

小強最容易出現在邊界

軟體測試的啟示：

Bug 最容易出現在：
  - 邊界值
  - 臨界點
  - 轉換點
    ↓ 所以
優先測試邊界

實際案例

案例1：條件判斷錯誤

正確程式：

if (x >= 10) {
    doSomething();
}

錯誤程式：

if (x == 10) {  // 漏掉 >
    doSomething();
}

測試分析：

等價劃分測試：
  測 9：  ✓ 正確判定為 False
  測 10： ✓ 正確判定為 True
    ↓ 問題
  測 11： ✗ 應為 True，實為 False
    ↓ 發現
只有測邊界"外"的值才能發現

結論：

二值法（測 9, 10）無法發現 三值法（測 9, 10, 11）才能發現

案例2：等於號遺漏

正確程式：

if (x > 10) {
    doSomething();
}

錯誤程式：

if (x >= 10) {  // 多了 =
    doSomething();
}

測試發現：

只有測試邊界值 10 才能發現差異

04 10.2 邊界值分析核心概念 ⭐⭐⭐

定義 ⭐⭐⭐ 必背

邊界值分析（Boundary Value Analysis, BVA）：

基於等價劃分的延伸技術 專注於測試輸入值的邊界點

與等價劃分的關係

必要條件：

先做等價劃分
  ↓ 找出邊界
才能做邊界值分析

為什麼？

邊界值分析需要知道：
  - 區間在哪裡
  - 邊界在哪裡
    ↓ 這些資訊來自
  等價劃分的結果

適用範圍

適用：

✓ 有序的序列
✓ 數值範圍
✓ 可排序的資料
  
例如：
  - 年齡：20-100
  - 溫度：0-100
  - 金額：1000-10000

不適用：

✗ 顏色選擇（紅、綠、藍）
✗ 性別（男、女）
✗ 狀態（開、關）
  
原因：無法排序

05 10.3 二值法（Two-Value BVA）⭐⭐⭐

定義與取值規則

二值法（2-Value BVA）：

測試邊界值及其最接近的鄰居值

取值口訣 ⭐⭐⭐ 必背

記憶口訣：
比小的小
比大的大

說明：

  • 比小的小：比最小值還小的值
  • 比大的大：比最大值還大的值

完整範例：年齡 20-100

步驟1：等價劃分

無效類別1：{..., 18, 19}     < 20
有效類別：  {20, 21, ..., 100}  20-100
無效類別2：{101, 102, ...}    > 100

步驟2：找邊界值

邊界值識別：

第一個邊界：20
  - 最小值
  
第二個邊界：100
  - 最大值

步驟3：二值法取值

取值方式：

邊界值1（20）：
  ✓ 20        （最小值）
  ✓ 19        （比最小值小）

邊界值2（100）：
  ✓ 100       （最大值）
  ✓ 101       （比最大值大）

測試案例：

TC1：19  → 應拒絕（無效）
TC2：20  → 應接受（有效）
TC3：100 → 應接受（有效）
TC4：101 → 應拒絕（無效）

覆蓋分析：

有效類別：覆蓋（20, 100）
無效類別1：覆蓋（19）
無效類別2：覆蓋（101）
  ↓
所有類別都有覆蓋 ✓

重要特性

特性1：固定數量

規律：

一個區間（兩個邊界）
  ↓ 二值法
取 4 個值

公式：

測試值數量 = 邊界數 × 2
  
一個區間：2 個邊界 → 4 個值
兩個區間：4 個邊界 → 8 個值

特性2：取值確定

與等價劃分的差異：

等價劃分：
  同類可取任意值
  每人取值可能不同

邊界值分析：
  必須取邊界值
  每人取值必須相同

例子：

區間：20-100

等價劃分（有效類別）：
  可取：20, 30, 50, 75, 100...
  ↓ 都可以

邊界值分析：
  必須取：20, 100
  ↓ 固定

06 10.4 三值法（Three-Value BVA）⭐⭐⭐

定義與取值規則

三值法（3-Value BVA）：

測試邊界值及其兩側的鄰居值

取值口訣 ⭐⭐⭐ 必背

記憶口訣：
邊界左右兩邊都要取

說明：

  • 每個邊界值
  • 比它大的值
  • 比它小的值

完整範例：年齡 20-100

三值法取值

取值方式：

邊界值1（20）：
  ✓ 19        （比最小值小）
  ✓ 20        （最小值）
  ✓ 21        （比最小值大）

邊界值2（100）：
  ✓ 99        （比最大值小）
  ✓ 100       （最大值）
  ✓ 101       （比最大值大）

測試案例：

TC1：19  → 應拒絕
TC2：20  → 應接受
TC3：21  → 應接受
TC4：99  → 應接受
TC5：100 → 應接受
TC6：101 → 應拒絕

與二值法的比較

數量對比：

二值法：4 個值
三值法：6 個值
  ↓ 多了
2 個值（21, 99）

偵錯能力：

二值法：
  可發現基本邊界錯誤
  
三值法：
  可發現更多邊界錯誤
  偵錯能力更強 ✓

使用時機：

官方建議：
  大部分情況：二值法
  高風險情況：三值法

為什麼三值法更強？

程式碼錯誤範例：

正確：x >= 10
錯誤：x == 10

測試比較：

二值法（9, 10）：
  9 → False（正確）
  10 → True（正確）
    ↓ 無法發現錯誤

三值法（9, 10, 11）：
  9 → False（正確）
  10 → True（正確）
  11 → False（錯誤！應為True）
    ↓ 發現錯誤！

結論：

三值法測邊界"內側" 增加發現缺陷機率

07 10.5 精確度的重要性 ⭐⭐⭐

精確度定義

什麼是精確度（Precision）：

數值的最小增減單位

常見精確度：

整數：精確度 = 1
  例：19, 20, 21

一位小數：精確度 = 0.1
  例：19.9, 20.0, 20.1

兩位小數：精確度 = 0.01
  例：19.99, 20.00, 20.01

⚠️ 不能用加減1 ⭐⭐⭐

常見錯誤：

✗ 錯誤做法：
  最小值 - 1
  最大值 + 1
  
為什麼錯？
  → 假設精確度為 1
  → 但實際可能不是

正確做法：

✓ 正確做法：
  依精確度調整
  
例：精確度 0.1
  最小值 20.0
    → 比它小：19.9
  最大值 25.0
    → 比它大：25.1

精確度應用範例

範例1：精確度 0.1

需求：

範圍：0.5 到 25
精確度：0.1

二值法取值：

最小值：0.5
  → 比它小：0.5 - 0.1 = 0.4 ✓

最大值：25
  → 比它大：25 + 0.1 = 25.1 ✓

測試案例：

TC1：0.4
TC2：0.5
TC3：25
TC4：25.1

範例2：精確度 1

需求：

範圍：20 到 100
精確度：1（整數）

二值法取值：

最小值：20
  → 比它小：20 - 1 = 19 ✓

最大值：100
  → 比它大：100 + 1 = 101 ✓

範例3：混合精確度陷阱

情境：

有人錯誤記憶：
  永遠用 ±1
  
問題：
  精確度 0.1 時
  最小值 0.5
  減 1 = -0.5 ✗
    ↓ 錯誤
  應該減 0.1 = 0.4 ✓

考試重點：

題目會明確給精確度 沒給就視為 1 絕對不要死背 ±1

08 10.6 邊界值分析覆蓋率 ⭐⭐⭐

覆蓋率公式 ⭐⭐⭐ 必背（v4.0.1 更正）

基本公式：

BVA 覆蓋率 = 已測試的邊界值數 / 總邊界值數 × 100%

⚠️ v4.0.1 重要說明：

分母：邊界值數量（不是測試值數量！）
分子：已覆蓋的邊界值數量

邊界值 = 區間的最小值與最大值
  例：20-100 有 2 個邊界值（20 和 100）

二值法覆蓋率計算

範例1：單一區間

區間：20-100

邊界值識別（分母）：

邊界值1：20（最小值）
邊界值2：100（最大值）
  ↓ 共 2 個邊界值 = 分母

二值法取值方式：

邊界值1（20）取 2 個測試值：
  - 19（比最小值小）
  - 20（最小值本身）

邊界值2（100）取 2 個測試值：
  - 100（最大值本身）
  - 101（比最大值大）
  
共需 4 個測試值達到 100% 覆蓋

計算範例：

測試案例：20, 100
  ↓ 覆蓋
  邊界值 20 ✓（測了20）
  邊界值 100 ✓（測了100）
  
覆蓋率 = 2/2 = 100%

⚠️ 注意：這是「邊界值覆蓋」
   但未完整覆蓋邊界外的值（19, 101）

完整覆蓋（建議做法）：

測試案例：19, 20, 100, 101
  ↓ 覆蓋
  邊界值 20 ✓（測了19和20）
  邊界值 100 ✓（測了100和101）
  
邊界值覆蓋率 = 2/2 = 100%
測試值覆蓋率 = 4/4 = 100%（完整二值法）

三值法覆蓋率計算

覆蓋項目

三值法覆蓋項目：

邊界值1（20）→ 產生 3 個覆蓋項目：
  - 19（比它小）
  - 20（邊界值）
  - 21（比它大）

邊界值2（100）→ 產生 3 個覆蓋項目：
  - 99（比它小）
  - 100（邊界值）
  - 101（比它大）
  
總覆蓋項目：6 個

完整覆蓋：

測試案例：19, 20, 21, 99, 100, 101
  ↓
覆蓋率 = 6/6 = 100%

多區間覆蓋率

範例：成績分級

需求：

0-50：  不及格
51-60： 及格
61-70： 滿意
71-80： 良好
81-90： 優秀
91-100：卓越

邊界值：

0, 50, 51, 60, 61, 70, 71, 80, 81, 90, 91, 100
共 12 個邊界值

二值法測試：

需要覆蓋的項目：12 個（邊界值本身）
加上鄰居值

實際上：
  因為鄰居值可能是另一個邊界值
  總覆蓋項目 = 12 個

已有測試案例：

TC1：91 → 卓越
TC2：50 → 不及格
TC3：81 → 優秀
TC4：60 → 及格
TC5：70 → 滿意
TC6：80 → 良好

覆蓋分析：

覆蓋的邊界值：
  50, 60, 70, 80, 81, 91
  共 6 個
  
覆蓋率 = 6/12 = 50%

09 10.7 邊界值分析實戰設計 ⭐⭐⭐

設計流程

標準流程

步驟1：等價劃分

確定有效與無效類別

步驟2：識別邊界值

找出每個區間的
  - 最小值
  - 最大值

步驟3：選擇方法

決定用二值法或三值法

步驟4：取值

依選定方法取邊界值及鄰居值

步驟5：設計測試案例

為每個取值設計測試案例
包含期望結果

實戰範例1：溫度控制

需求：

溫度範圍：0°C 到 2°C
  - 範圍內：顯示"溫度正常"
  - 太低：顯示"溫度過低"
  - 太高：顯示"溫度過高"
精確度：1°C（整數）

二值法設計：

步驟1：等價劃分

無效1：{..., -2, -1}     < 0
有效：  {0, 1, 2}         0-2
無效2：{3, 4, ...}       > 2

步驟2：邊界值

邊界1：0
邊界2：2

步驟3：二值法取值

邊界1（0）：
  -1（比它小）
  0 （邊界值）

邊界2（2）：
  2（邊界值）
  3（比它大）

測試案例：

TC1：-1 → "溫度過低"
TC2：0  → "溫度正常"
TC3：2  → "溫度正常"
TC4：3  → "溫度過高"

三值法設計（高風險情況）：

額外取值：

邊界1（0）再加：
  1（比它大）

邊界2（2）再加：
  1（比它小）

完整測試案例：

TC1：-1 → "溫度過低"
TC2：0  → "溫度正常"
TC3：1  → "溫度正常"  ← 新增
TC4：2  → "溫度正常"
TC5：3  → "溫度過高"

注意：

1 被測試兩次
  → 分別來自兩個邊界
  → 但只算一個覆蓋項目

實戰範例2：密碼長度

需求：

密碼長度：6 到 12 個字元
精確度：1（整數）

二值法 → 三值法升級

已達成二值法 100%：

測試案例：
  0（太短）
  5（太短）
  6（正常）
  12（正常）
  13（太長）

分析：

二值法覆蓋項目：
  5, 6, 12, 13（共 4 個）
  
已覆蓋：0, 5, 6, 12, 13
  = 5/4？錯誤！
  
實際：
  0 不在二值法範圍內
  真正覆蓋：5, 6, 12, 13
  覆蓋率 = 4/4 = 100% ✓

升級到三值法：

三值法額外需要：
  7（6的右鄰居）
  11（12的左鄰居）
  
新增測試案例：
  TC6：7  → 應接受
  TC7：11 → 應接受

三值法完整測試：

5, 6, 7, 11, 12, 13（共 6 個）
覆蓋率 = 6/6 = 100%

10 10.8 精確度實戰案例 ⭐⭐⭐

案例1：藥品劑量

需求：

劑量範圍：0.5 mg 到 25 mg
精確度：0.1 mg

錯誤做法 ✗：

取值：
  0.5 - 1 = -0.5
  25 + 1 = 26
    ↓ 問題
精確度是 0.1，不是 1

正確做法 ✓：

二值法取值：
  0.5 - 0.1 = 0.4 ✓
  0.5
  25
  25 + 0.1 = 25.1 ✓

案例2：整數陷阱

需求：

範圍：101 到 199
精確度：1（整數）

看似簡單：

邊界：101, 199
  
二值法：
  100（101-1）
  101
  199
  200（199+1）

但要理解本質：

不是死背 ±1
而是理解：
  找"比邊界值小但最接近"的值
  找"比邊界值大但最接近"的值

案例3：考試陷阱

題目：

範圍：0.5 到 25
精確度：0.1

已達成二值法 100%，問：
  要從二值法升級到三值法
  還需測試哪些值？

分析：

二值法：0.4, 0.5, 25, 25.1（4個）
  
三值法需要：
  0.4, 0.5, 0.6（邊界1附近）
  24.9, 25, 25.1（邊界2附近）
  共 6 個
  
額外需要：
  0.6（0.5的右鄰居）
  24.9（25的左鄰居）

答案：

還需測試：0.6, 24.9
（或其他符合的組合）

11 10.9 常見考題類型與解法 ⭐⭐⭐

題型1：覆蓋率計算（K3）

標準題型：

Q: 範圍 20-100，
   已測試 20, 100，
   二值法覆蓋率為？

解題步驟：

1. 列出二值法所有覆蓋項目
   19, 20, 100, 101（4個）

2. 檢查測試案例覆蓋了哪些
   20, 100（2個）

3. 計算
   2/4 = 50%

題型2：區間選擇（K3）

題目：

Q: 下列哪個選項符合二值法？
   範圍：150 到 200
   
A. 149, 200
B. 150, 200, 201
C. 149, 150, 200, 201
D. 150, 175, 200

解法：

二值法需要：
  149（比150小）
  150（邊界）
  200（邊界）
  201（比200大）
  
答案：C（4個值齊全）

題型3：三值法補充（K3）

題目：

Q: 已達成二值法100%（測試：5,6,12,13）
   要升級到三值法，還需測試哪些值？
   範圍：6-12

解法：

三值法需要：
  5, 6, 7（邊界1附近）
  11, 12, 13（邊界2附近）
  
已有：5, 6, 12, 13
  
還需：7, 11

答案：7, 11

12 10.10 邊界值分析進階技巧

技巧1：邊界值重複

情境：

區間1：0-50
區間2：51-100
  ↓ 邊界值
0, 50, 51, 100

取值時：

50 是區間1的最大值
51 是區間2的最小值
  ↓ 兩個都要測
  ↓ 但不重複計算覆蓋項目

技巧2：開區間與閉區間

開區間：不包含邊界

範圍：0 < x < 10
  ↓ 邊界不在有效類別
  
有效：{1, 2, ..., 9}
無效1：{..., -1, 0}
無效2：{10, 11, ...}
  
邊界值：1, 9

閉區間：包含邊界

範圍：0 ≤ x ≤ 10
  ↓ 邊界在有效類別
  
有效：{0, 1, ..., 10}
無效1：{..., -2, -1}
無效2：{11, 12, ...}
  
邊界值：0, 10

技巧3：非數字序列

日期範圍：

2024/01/01 到 2024/12/31

邊界值：
  2024/01/01
  2024/12/31
  
二值法：
  2023/12/31（比最小日期小）
  2024/01/01
  2024/12/31
  2025/01/01（比最大日期大）

字串排序（較少見）：

範圍：A 到 Z

概念上的邊界：
  A（第一個）
  Z（最後一個）
  
但實務較少用BVA測試字串

13 10.11 等價劃分 vs 邊界值分析 ⭐⭐⭐

核心差異

比較項目等價劃分邊界值分析
